README for JASA 15/3/2002

The Auction Simulator API allows researchers in experimental economics to write high-performance trading simulations using a number of different auction protocols. The software also provides base classes for implementing simple adaptive trading agents.

The system is based on the 4-heap algorithm, described in the paper

"Flexible Double Auctions for Electronic Commerce: Theory and Implementation" by Wurman, Walsh amnd Wellman 1998.

The system implements various experiments, agents and learning agents described elsewhere in the experimental economics literature. The documentation provides citation details where appropriate.

Documentation

The javadoc documentation can be found here.

Required libraries

JUnit (Optional) required for running the unit tests

ECJ (Optional) Required for running the genetic-programming experiments

Examples

The following examples demonstrate the use of the auction simulator:

uk.ac.liv.auction.DCSimulation
uk.ac.liv.auction.MarketSimulation
uk.ac.liv.auction.SupplyRingSimulation

Each of these examples can be run as an independent application. The file auction.params can be used to initialise ECJ to run the genetic programming experiments.
