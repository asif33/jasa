The Auction Simulator API allows researchers in agent-based computational economics to write high-performance trading simulations using a number of different auction protocols. The software also provides base classes for implementing simple adaptive trading agents.

    • About

    Features:

  • High performance (suitable for use in evolutionary-computing experiments)

  • Highly extensible architecture

  • Highly configurable - experiments can easily be configured from parameter files

  • Optional real-time graphing of simulation as it progresses

  • Implementations of existing agents, learning algorithms, experiments and auction types described in the computationl economics literature.

  • Experimental support for the JADE agent platform.

    • Development Status

    This package is currently at alpha. This code is not stable or fully tested. Please report any bugs, issues or suggestions to Steve Phelps.

    Prerequisites

  • Java JVM 1.4.0 or later

  • PooliT version 2.0 or later

  • ECJ version 10 or later

  • Log4j

  • JSci version 0.91 or later

  • RngPack version 1.1a or later

  • JADE version 3.0b1 or later - (Optional) required for running simulations under the JADE agent platform

  • JUnit version 3.81 or later - (Optional) required for running the unit tests

    • Optional libraries are required for compilation, but are not required for running JASA if the indicated functionality is not being used. The core libraries can be downloaded from SourceForge.

    Installation

  • Unpack the distribution into a directory. The instructions refer to this directory as JASADIR from here on.

  • Obtain all of the libraries listed in the prerequisites section, and place them in the directory JASADIR/jasa/lib.

  • Set your CLASSPATH environment variable to include the file JASADIR/jasa/jasa.jar, and all of the libraries in the directory JASADIR/jasa/lib. On linux, use the command:

    export CLASSPATH=$CLASSPATH:JASADIR/jasa/jasa.jar:JASADIR/jasa/lib/ecj:JASADIR/jasa/lib/log4j.jar: JASADIR/jasa/lib/JSci/classes/core.jar:JASADIR/jasa/lib/JSci/classes/sci.jar: JASADIR/jasa/lib/JSci/classes/xtra.jar:JASADIR/jasa/lib/poolit/poolit.jar

    • Running the examples

    The examples/ directory contains several parameter files for different example experiments. These can be run using the uk.ac.liv.auction.MarketSimulation application, e.g.

    java uk.ac.liv.auction.MarketSimulation JASADIR/jasa/examples/example1.params

    Compiling the code

    JASA ships with a build.xml file that can be used with Apache Ant to build the project. The default settings in build.xml expect to find the various third-party class libraries used by JASA either in the CLASSPATH or in the directory jasa/lib, however alternative locations can be specified by creating a file build.ant.properties.

    Acknowledgements

    This work was partially supported by the EU IST Programme through the SLIE Project.

    The system is based on the 4-heap algorithm, described in the paper

    "Flexible Double Auctions for Electronic Commerce: Theory and Implementation" by Wurman, Walsh and Wellman 1998.

    JASA makes use of the Mersenne Twister PRNG. Full details of the Mersenne Twister algorithm can be found in:

    Makato Matsumoto and Takuji Nishimura, "Mersenne Twister: A 623-Dimensionally Equidistributed Uniform Pseudo-Random Number Generator", in ACM Transactions on modeling and Computer Simulation, Vol. 8, No. 1, January 1998, pp 3--30.
