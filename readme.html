The Auction Simulator API allows researchers in experimental economics to write high-performance trading simulations using a number of different auction protocols. The software also provides base classes for implementing simple adaptive trading agents.

The system is based on the 4-heap algorithm, described in the paper

"Flexible Double Auctions for Electronic Commerce: Theory and Implementation" by Wurman, Walsh amnd Wellman 1998.

The system implements various experiments, agents and learning agents described elsewhere in the experimental economics literature. The documentation provides citation details where appropriate.

The system currently supports parameterisable auctioneer classes for the following auction types:

  • Multi-Unit Continuous Double Auction
  • Multi-Unit Ascending Auction

    • The system is designed so that introducing new classes of auctioneers should be relatively straightforward.

    Required libraries

    PooliT

    ECJ

    JADE (Optional) required for running simulations under the JADE agent platform

    JUnit (Optional) required for running the unit tests

    Skij (Optional) required for manipulating koza trees as scheme s-expressions

    Optional libraries are required for compilation, but are not required for running JASA if the indicated functionality is not being used.

    Running

    Ensure that all required libraries, including the jasa.jar file, are present in your CLASSPATH environment variable, and then run the class uk.ac.liv.auction.MarketSimulation, giving the name of a parameter file as the first argument.

    Examples

    The examples/ directory contains several parameter files for different example experiments. These can be run using the uk.ac.liv.auction.MarketSimulation application, e.g.

    java uk.ac.liv.auction.MarketSimulation examples/example1.params

    Compiling

    JASA ships with a build.xml file that can be used with Apache Ant to build the project. The default settings in build.xml expect to find the various third-party class libraries used by JASA in the directory /usr/local/java/lib, however alternative locations can be specified by creating a file build.ant.properties.
