    • About

    JASA is a high-performance auction simulator written in JAVA. It is designed for performing experiments in agent-based computational economics.

    Features:

  • High performance

  • Integration with ECJ for performing experiments involving evolutionary computing or genetic programming.

  • Integration with RePast for visualising and controlling simulations.

  • Highly extensible architecture

  • Highly configurable - experiments can easily be configured from parameter files

  • Pluggable pseudo-random number generators using the CERN colt library

  • Implementations of existing agents, learning algorithms, experiments and auction types described in the computational economics literature.

    • Development Status

    This package is currently at alpha. This code is not stable or fully tested. Please report any bugs, issues or suggestions to Steve Phelps.

    Prerequisites

  • Java JVM 1.4.0 or later
  • Apache Ant version 1.6.2 or later.

    Dependencies

    JASA also ships with the following third-party libraries, all of which are available under open-source licenses:

  • Log4j version 1.2.8
  • JUnit version 3.81
  • Scheme package version 1.1
  • RePast version 3.0
  • Commons Collections version 3.1

    • Installation

  • Simply unpack the distribution into a directory. The instructions refer to this directory as JASADIR from here on.

    • Compiling the code

    JASA ships with a build.xml file that can be used with Apache Ant to build the project. The default settings in build.xml expect to find the various third-party class libraries used by JASA either in the CLASSPATH or in the directory jasa/lib, however alternative locations can be specified by creating a file build.ant.properties.

    Running the examples

    The examples/ directory contains several parameter files for different example experiments. These can be executed using Apache Ant by specifying the target run, and defining a property params specifying the parameter file you want to use for the simulation; e.g. cd into the directory JASADIR and execute the following command:

    ant run -Dparams=examples/example2.params

    Acknowledgements

    This work was partially supported by the EU IST Programme through the SLIE Project.

    The system is based on the 4-heap algorithm, described in the paper

    "Flexible Double Auctions for Electronic Commerce: Theory and Implementation" by Wurman, Walsh and Wellman 1998.

    JASA makes use of the Mersenne Twister PRNG. Full details of the Mersenne Twister algorithm can be found in:

    Makato Matsumoto and Takuji Nishimura, "Mersenne Twister: A 623-Dimensionally Equidistributed Uniform Pseudo-Random Number Generator", in ACM Transactions on modeling and Computer Simulation, Vol. 8, No. 1, January 1998, pp 3--30.
