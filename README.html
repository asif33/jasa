    • About

    JASA is a high-performance auction simulator written in JAVA. It is designed for performing experiments in agent-based computational economics.

    Development Status

    This package is currently at alpha. This code is not stable or fully tested. Please report any bugs, issues or suggestions to Steve Phelps.

    Prerequisites

  • Java JVM 1.6.0 or later
  • Apache Ant version 1.6.2 or later.

    Dependencies

    JASA is built on top of the JABM framework which must be installed before using JASA.

    Installation

    JASA can be run either using either Apache Ant or the Java Eclipse IDE.

    Running JASA using the Eclipse IDE

  • See the Eclipse Web site for instructions on installing and configuring Eclipse.

  • Select the 'Import' option from the 'File' menu and choose 'Existing projects into Workspace' from the 'General' category. Choose the 'select archive file' option and then browse to the location where you downloaded the distribution .tar.gz file (eg 'jasa-0.36.tar.gz').

  • JASA has two main classes: uk.ac.liv.auction.RepastMarketSimulation and uk.ac.liv.auction.MarketSimulation. The former can be used to run experiments interactively with visualisation provided by Repast, whereas the latter can be used to run a series of experiments in batch-mode. Both applications take a single argument specifying the parameters to use for each experiment. The distribution ships with several example parameter files in the 'examples' directory. To run the experiment for example2:

  • Open the run configuration dialog in Eclipse using the 'Run/Run Configurations..' menu;
  • Create a new 'java application' configuration and specify uk.ac.liv.auction.RepastMarketSimulation as the main class;
  • Specify examples/example2.params for the program arguments in the Arguments tab.

    • Running JASA using Apache Ant

  • Simply unpack the distribution into a directory. The instructions refer to this directory as JASADIR from here on.

    • Compiling the code

    JASA ships with a build.xml file that can be used with Apache Ant to build the project. The default settings in build.xml expect to find the various third-party class libraries used by JASA either in the CLASSPATH or in the directory jasa/lib, however alternative locations can be specified by creating a file build.ant.properties.

    Running the examples

    The examples/ directory contains several parameter files for different example experiments. These can be executed using Apache Ant by specifying the target run, and defining a property params specifying the parameter file you want to use for the simulation; e.g. cd into the directory JASADIR and execute the following command:

    ant run -Dparams=examples/example2.params

    To run a batch experiment, use the 'batchrun' command, eg:

    ant batchrun -Dparams=examples/example1.params

    Alternatively, for maximum performance, set your CLASSPATH environment to include all the libraries used by JASA, and then execute the following command:

    java -server uk.ac.liv.auction.MarketSimulation examples/example1.params

    The -server option tells the JVM to use the server version of the just-in-time (JIT) compiler, which will perform many more optimizations than the standard JIT.

    Parameter files

    JASA uses the same framework as ECJ for configuring experiments. See ECJ's primer on parameter files.

    The system is based on the 4-heap algorithm, described in the paper

    "Flexible Double Auctions for Electronic Commerce: Theory and Implementation" by Wurman, Walsh and Wellman 1998.

    JASA makes use of the Mersenne Twister PRNG. Full details of the Mersenne Twister algorithm can be found in:

    Makato Matsumoto and Takuji Nishimura, "Mersenne Twister: A 623-Dimensionally Equidistributed Uniform Pseudo-Random Number Generator", in ACM Transactions on modeling and Computer Simulation, Vol. 8, No. 1, January 1998, pp 3--30.
